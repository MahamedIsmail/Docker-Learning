About the Project

"Visit where it counts"

Welcome to my Visitor Counter App. What started as a simple tracking tool evolved into a robust exercise in modern infrastructure, high-performance scaling, and DevOps best practices.

🛠️ Core Tech Stack

  • Flask & Redis: Real-time data processing and storage.
  • Google Maps API: Geospatial visualization of visitor traffic.
  • Tailwind CSS: A clean, utility-first responsive interface.
  • IP Geolocation: Live tracking of city-level visitor data.

🚀 Infrastructure & DevOps

  • Docker Compose: Full orchestration of the app ecosystem.
  • Custom Redis: Hand-built Redis image with optimized configs.
  • Nginx Scaling: Reverse proxy setup for real-time load handling.
  • Multi-stage Builds: Optimized Dockerfiles for lightweight deployments.

🔧 Technical Notes

I built this app with a focus on reliability and performance. To ensure the application never breaks due to external updates, I strictly froze all dependencies in my requirements; this guarantees the Docker image remains stable and functional years from now.

Furthermore, I implemented multi-stage Docker builds. By separating the build environment from the final runtime, I significantly reduced the image size, resulting in faster deployment times and a smaller security attack surface.

Combined with an Nginx configuration for horizontal scaling, this project is ready for real-world traffic.
